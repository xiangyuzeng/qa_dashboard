# AI 食品安全技术信息及舆情监控模型需求说明（v1）

> 原始（总体）需求。细化版见 `spec_v2.md`；两者冲突时以 v2 为准。

## 一、项目目标
建立一个基于 AI 的食品安全技术信息及舆情监控模型，用于持续监控美国联邦层面及各州、地方政府公开渠道中的食品安全、法规标准、监管执法、餐饮门店检查、产品召回、监督抽检、突发食品安全事件及相关产品负面舆情信息。

模型应能够自动抓取、筛选、分类、摘要、翻译和汇总相关信息，并按月生成食品安全技术信息及舆情监控月报。对于重大食品安全事件、重大法规变化、严重监管处罚或与甲方业务高度相关的负面舆情，模型应具备风险识别和预警能力。

## 二、监控范围
### 1. 地理范围
- 美国联邦层面；
- 美国各州；
- 重点城市、县及地方卫生部门；
- 甲方门店、仓库、供应链、进口业务或销售区域涉及的重点地区。

### 2. 监管机构范围
模型应覆盖以下类型的信息来源：FDA；USDA / FSIS；CDC；EPA；FTC；CPSC；CBP；APHIS；OSHA；各州卫生部门、农业部门、消费者保护部门；各市、县、地方 DOH 餐饮卫生检查数据库；其他与食品安全、进口合规、标签、广告宣称、公共卫生、产品安全相关的政府及权威机构。

## 三、监控内容
1. **政策法规与标准变动** — 食品安全、食品标签、进口合规、食品添加剂、过敏原、营养标签、广告宣称、包装材料、食品接触材料、餐饮门店卫生、公共卫生及职业安全等相关法律法规、指南文件、标准、征求意见稿、监管解释及执法政策变化。
2. **政府监管动态** — 政府公告、执法通报、检查重点、进口预警、警告信、监管函、执法行动、行业专项整治、合规指南及监管趋势。
3. **产品召回及安全警示** — FDA、USDA、FSIS、州政府及企业主动发布的食品召回、过敏原误标、微生物污染、异物污染、化学污染、标签错误、包装缺陷及其他产品安全警示。
4. **监督抽检及风险监测结果** — 联邦、州及地方政府发布的食品抽检、微生物检测、过敏原检测、污染物检测、农兽药残留、重金属、食品添加剂、标签合规、异物及其他风险监测结果。
5. **餐饮门店 DOH 检查及扣分项** — 从当地 DOH 或地方卫生部门数据库中拉取餐饮门店卫生检查数据：门店名称、地址、所属城市/州/县、检查日期、检查类型、检查结果、等级或评分、扣分总数、具体违规项、违规代码、违规描述、严重违规项、复查或整改要求、原始数据链接。
   - 扣分项标准化分类示例：食品温度控制；交叉污染；员工个人卫生；洗手设施；清洁消毒；虫害控制；食品储存；设备设施维护；三槽水池/洗碗机；标签及日期标识；垃圾管理；证照及文件记录；其他食品安全或卫生管理问题。
   - 高风险扣分项示例：冷藏温度超标、热食保温不足、生熟交叉污染、洗手设施不可用、鼠蟑活动迹象、污水倒流、食品来源不明、过期食品使用等。
6. **重大突发食品安全事件** — 食源性疾病暴发、重大污染事件、大规模召回、供应链风险、进口受阻、监管调查、消费者集中投诉等。
7. **相关产品负面报道及舆情** — 新闻媒体、行业媒体、政府公开信息、消费者投诉平台、社交媒体公开信息中，涉及甲方相关产品、同类产品、竞品品牌、重点供应商、关键原料、食品添加剂、包装材料、食品接触材料、餐饮门店卫生、标签宣称、消费者食品安全投诉、品牌声誉风险的负面报道。

## 四、AI 模型核心功能需求
1. **多源数据抓取** — 从政府网站、监管数据库、新闻媒体、行业网站、公开数据库及地方 DOH 系统自动或半自动抓取。结构化数据库（如 DOH 餐饮检查数据库）优先通过 API、公开数据集、可下载文件或网页数据表拉取；非结构化网页应识别标题、发布时间、来源、正文、链接、涉及产品、涉及企业、风险类型和监管动作。
2. **信息去重与有效筛选** — 去重、过滤、相关性判断。筛选逻辑重点考虑：是否涉及甲方经营地区/产品类别/供应商/竞品/同类产品；是否涉及食品安全、法规合规或品牌风险；是否属于重大监管变化或高风险事件；来源是否权威可靠；发布时间是否在监控周期内。
3. **自动分类** — 法规标准；监管动态；召回事件；食品安全事件；监督抽检；DOH 检查/扣分项；负面舆情；供应链风险；标签/广告宣称风险；进口合规风险；其他。
4. **中文摘要与翻译** — 英文原文转中文摘要，准确、客观、简洁。每条摘要至少包括：事件或法规核心内容；涉及产品/企业/地区；主要风险点；监管机构或信息来源；对甲方可能产生的影响。
5. **风险评级** — 高风险 / 中风险 / 低风险 / 信息参考（依据事件严重性、相关性、监管影响和舆情影响）。
6. **关键词与业务关联匹配** — 维护关键词库：甲方品牌名称；门店名称及地址；供应商名称；产品名称；原料名称；食品添加剂名称；包装及食品接触材料；竞品品牌；高风险食品安全关键词；法规及监管关键词。支持定期更新。
7. **DOH 扣分项趋势分析** — 单店历史扣分趋势；同一区域门店扣分趋势；高频违规项统计；严重违规项识别；重复违规项识别；竞品或行业门店对比；当地 DOH 执法重点变化；甲方门店自查重点风险提示。
8. **月报自动生成** — 每个自然月结束后第 5 个工作日前发送至甲方指定邮箱。月报字段：序号、中文标题、信息类别、来源、发布日期、原文摘要、原文链接、风险等级、相关性说明；涉及 DOH 检查信息增加：门店名称、地址、所属地区、检查日期、检查类型、检查结果/等级、扣分总数、主要扣分项、严重违规项、原文链接、风险提示。
9. **重大事件预警** — 触发条件：涉及甲方或供应商的监管处罚/召回/负面报道；同类产品重大食品安全事件；高度相关的食源性疾病暴发；影响进口/标签/销售/门店运营的新法规；重点地区 DOH 监管政策或扣分标准明显变化；甲方或竞品门店严重 DOH 扣分项；可能引发品牌声誉风险的媒体或消费者负面报道。预警内容：事件标题、发布时间、信息来源、原文链接、事件摘要、风险等级、与甲方业务的相关性、建议关注事项。

## 五、输出要求
- 中文表达清晰、准确、专业；
- 所有信息必须保留原文来源和链接；
- 不得编造来源、发布日期或监管结论；
- 摘要应忠实于原文；
- 风险评级应基于明确规则；
- 月报格式应统一，便于归档和追溯；
- 支持 Excel、Word、PDF 或网页仪表盘等输出形式；
- 支持通过电子邮件自动发送至甲方指定邮箱。

## 六、数据字段要求
每条监控信息建议至少包含以下字段：

| 字段 | 说明 |
|---|---|
| 中文标题 | AI 生成的中文标题 |
| 英文原标题 | 原始英文标题 |
| 信息类别 | 法规、召回、DOH 检查、舆情等 |
| 来源机构 / 媒体 | 原始信息发布方 |
| 发布日期 | 原文发布日期 |
| 抓取日期 | AI 系统抓取日期 |
| 适用地区 | 联邦、州、市、县 |
| 涉及产品 | 涉及的食品、原料或包装材料 |
| 涉及企业 | 涉及品牌、供应商、门店或生产商 |
| 原文摘要 | 中文摘要 |
| 原文链接 | 可追溯链接 |
| 风险等级 | 高 / 中 / 低 / 信息参考 |
| 相关性说明 | 与甲方业务的关联 |
| 建议关注事项 | 后续建议或内部关注点 |

## 七、模型管理要求
- 支持人工复核机制，月报发送前可由乙方或甲方指定人员审核；
- 支持关键词库、地区范围、供应商名单、竞品名单和产品类别的维护；
- 保留历史监控记录，便于追溯；
- 支持结果导出；
- 支持按时间、地区、类别、风险等级、产品、企业检索；
- 定期优化筛选规则，降低漏报和误报。

## 八、服务交付要求
- 乙方基于 AI 模型持续监控，并按月交付监控月报；
- 月报于每个自然月结束后第 5 个工作日前发送至甲方指定邮箱；
- 高风险或紧急信息应在识别后及时发送专项预警，不限于月报周期。

## 九、责任边界
AI 模型输出内容用于食品安全、法规合规和舆情风险监控参考，不替代律师、法规顾问、监管机构或实验室的正式意见。涉及产品上市、标签修改、进口申报、监管应对、召回决策或重大经营决策的事项，甲方应结合专业法律、法规、技术或监管意见进行最终判断。
